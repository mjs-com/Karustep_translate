# TranslatorApp 開発計画書（ネイティブアプリ版）

## 1. 概要

本計画書は、**Phase 0の技術検証でWebアプリが不適と判断された場合**の代替案として、
**Electron/Tauri等を用いたネイティブアプリ**として開発する場合の計画書です。

> ⚠️ **前提条件**: Webアプリ版の計画書（`アプリ作成計画書.md`）のPOC-1（フットスイッチ対応）が
> No-Goと判定された場合に、本計画書に移行します。

### 1.1 ネイティブアプリを選択する条件

| POC結果 | 判定 | 対応 |
|---------|------|------|
| Chromeフォーカス中にフットスイッチのキーイベントが発火する | Go | Webアプリ版で開発 |
| Chromeがフォーカスを失うとキーイベントが発火しない | 許容可能 | Webアプリ版で開発（運用でカバー） |
| そもそもキーイベントが発火しない | No-Go | **本計画書（ネイティブアプリ版）で開発** |

### 1.2 Webアプリ版との比較

| 項目 | Webアプリ版 | ネイティブアプリ版 |
|------|------------|-------------------|
| **フレームワーク** | React + FastAPI | Electron/Tauri + React |
| **フットスイッチ** | KeyboardEvent（制限あり） | グローバルホットキー対応 |
| **音声ファイル** | 保存しない | ローカルPCに保存 |
| **文字起こし** | データベース保存 | ローカルテキストファイル |
| **要約結果** | データベース保存 | ローカルテキストファイル |
| **オフライン** | 不可 | 録音のみ可能 |
| **マルチPC** | どのPCからでもアクセス可能 | インストールしたPCのみ |
| **インストール** | 不要 | 必要 |
| **VoiceRecorder流用度** | 中（API呼び出し部分のみ） | **高（ファイル構造も流用）** |

## 2. 技術スタック

### 2.1 フレームワーク選定

| 選択肢 | 長所 | 短所 | 推奨度 |
|--------|------|------|--------|
| **Electron** | 実績豊富、npmエコシステム活用 | バイナリサイズ大（〜150MB） | ⭐⭐⭐ |
| **Tauri** | 軽量（〜10MB）、Rust製で高速 | 学習コスト高、実績少 | ⭐⭐ |
| **C# WPF（VoiceRecorder拡張）** | VoiceRecorderのコードを直接流用 | Windows専用 | ⭐⭐⭐⭐ |

**推奨: C# WPF（VoiceRecorder拡張）**

理由：
- VoiceRecorderの既存コードをほぼそのまま流用可能
- グローバルホットキー、NAudio録音など既に実装済み
- 翻訳機能の追加のみで対応可能

### 2.2 技術スタック（VoiceRecorder拡張版）

| レイヤー | 技術 | VoiceRecorderからの流用 |
|---------|------|------------------------|
| **UI** | WPF/XAML | ✅ MainWindow.xaml を拡張 |
| **録音** | NAudio | ✅ SoundRecorder完全流用 |
| **STT** | Azure Fast Transcription API | ✅ SpeechToText.cs完全流用 |
| **翻訳** | DeepL API | 🆕 新規実装 |
| **要約** | Azure OpenAI GPT-4o | ✅ SummarizeText.cs完全流用 |
| **ホットキー** | RegisterHotKey | ✅ MainWindow.xaml.cs流用 |
| **ファイル保存** | ローカルファイルシステム | ✅ RecordingSession.cs流用 |

### 5.3 TranslationService.cs（新規作成）

```csharp
// TranslationService.cs
using System;
using System.Net.Http;
using System.Text;
using System.Text.Json;
using System.Threading.Tasks;

namespace VoiceRecorder
{
    public class TranslationService
    {
        private static readonly HttpClient httpClient = new HttpClient
        {
            Timeout = TimeSpan.FromSeconds(30)
        };

        private const string DEEPL_ENDPOINT = "https://api.deepl.com/v2/translate";

        /// <summary>
        /// DeepL APIで翻訳を実行
        /// </summary>
        public static async Task<string> TranslateAsync(
            string text, 
            string sourceLang, 
            string targetLang)
        {
            string apiKey = CredentialsProvider.GetDeepLApiKey();
            
            var content = new FormUrlEncodedContent(new[]
            {
                new KeyValuePair<string, string>("text", text),
                new KeyValuePair<string, string>("source_lang", ConvertToDeepLCode(sourceLang)),
                new KeyValuePair<string, string>("target_lang", ConvertToDeepLCode(targetLang)),
            });

            httpClient.DefaultRequestHeaders.Clear();
            httpClient.DefaultRequestHeaders.Add("Authorization", $"DeepL-Auth-Key {apiKey}");

            var response = await httpClient.PostAsync(DEEPL_ENDPOINT, content);
            
            if (!response.IsSuccessStatusCode)
            {
                string error = await response.Content.ReadAsStringAsync();
                throw new Exception($"DeepL API Error: {response.StatusCode} - {error}");
            }

            string jsonResponse = await response.Content.ReadAsStringAsync();
            var json = JsonSerializer.Deserialize<JsonElement>(jsonResponse);
            
            return json.GetProperty("translations")[0].GetProperty("text").GetString() ?? "";
        }

        /// <summary>
        /// Azure STTの言語コードをDeepLコードに変換
        /// </summary>
        private static string ConvertToDeepLCode(string azureCode)
        {
            return azureCode switch
            {
                "ja-JP" => "JA",
                "en-US" => "EN-US",
                "en-GB" => "EN-GB",
                "zh-CN" => "ZH",
                "zh-TW" => "ZH",
                "ko-KR" => "KO",
                "es-ES" => "ES",
                "pt-BR" => "PT-BR",
                "vi-VN" => "VI",
                _ => azureCode.Split('-')[0].ToUpper()
            };
        }
    }
}
```

### 5.4 LanguageRoleMapper.cs（新規作成）

```csharp
// LanguageRoleMapper.cs
using System;

namespace VoiceRecorder
{
    public class LanguageRoleMapper
    {
        public string DoctorLanguage { get; }
        public string PatientLanguage { get; }

        public LanguageRoleMapper(string doctorLanguage, string patientLanguage)
        {
            DoctorLanguage = doctorLanguage;
            PatientLanguage = patientLanguage;
        }

        /// <summary>
        /// 検出された言語からロール（医師/患者）を判定
        /// </summary>
        public string GetRole(string detectedLanguage)
        {
            if (string.IsNullOrEmpty(detectedLanguage))
                return "unknown";

            // 完全一致
            if (detectedLanguage == DoctorLanguage)
                return "doctor";
            if (detectedLanguage == PatientLanguage)
                return "patient";

            // 言語ファミリーで判定（en-US と en-GB は同じ英語として扱う）
            string detectedBase = detectedLanguage.Split('-')[0];
            if (DoctorLanguage.StartsWith(detectedBase))
                return "doctor";
            if (PatientLanguage.StartsWith(detectedBase))
                return "patient";

            return "unknown";
        }

        /// <summary>
        /// ロールから翻訳先言語を取得
        /// </summary>
        public string GetTranslationTarget(string role)
        {
            return role switch
            {
                "doctor" => PatientLanguage,  // 医師の発言 → 患者の言語に翻訳
                "patient" => DoctorLanguage,  // 患者の発言 → 医師の言語に翻訳
                _ => DoctorLanguage
            };
        }
    }
}
```

## 8. 今回のまとめ

### ネイティブアプリ版の特徴

1. **VoiceRecorderのコードを最大限流用**
   - 録音機能（NAudio）、STT（SpeechToText.cs）、要約（SummarizeText.cs）は完全流用
   - グローバルホットキー対応でフットスイッチが確実に動作
   - 既存の安定したコードベースを活用

2. **追加実装が必要な機能**
   - TranslationService.cs（DeepL API呼び出し）
   - LanguageRoleMapper.cs（言語→ロール変換）
   - 言語設定UI
   - 2カラム表示UI

3. **データはローカルファイルに保存**
   - 音声ファイル、文字起こし、翻訳、要約をすべてローカル保存
   - VoiceRecorderと同じファイル構造を踏襲
   - オフラインでも録音可能（翻訳はオンライン時に実行）

### Webアプリ版との使い分け

| 状況 | 推奨版 |
|------|--------|
| フットスイッチがChromeで動作する | Webアプリ版 |
| 複数PCからアクセスしたい | Webアプリ版 |
| フットスイッチがChromeで動作しない | **ネイティブアプリ版** |
| オフラインでも使いたい | **ネイティブアプリ版** |
| VoiceRecorderの既存コードを活用したい | **ネイティブアプリ版** |
