# TranslatorApp 開発計画書 v2.1（Webアプリ版）

## 1. 概要

本計画書は、既存のVoiceRecorder（C# WPF Windowsアプリ）で確立された技術を流用し、新たに**医療通訳Webアプリ（TranslatorApp）**を開発するための詳細な設計・実装計画です。

> ⚠️ **注意**: 本計画書は**Webアプリ**として開発する場合の計画書です。
> Phase 0の技術検証（特にフットスイッチ対応）の結果、ネイティブアプリが必要と判断された場合は
> `アプリ作成計画書（ネイティブアプリの場合）.md` を参照してください。

### 1.1 プロジェクトの目的
- 外国人患者と日本人医師の会話をリアルタイムで**文字起こし**・**翻訳**し、チャット形式で表示する
- 診察終了後に会話ログからSOAP形式のカルテと英文診断書を自動生成する
- クリニックごとのデータ分離と、API使用量に基づく従量課金管理を実装する

### 1.2 Webアプリ vs ネイティブアプリの違い

| 項目 | Webアプリ（本計画書） | ネイティブアプリ |
|------|---------------------|-----------------|
| **音声ファイル** | 保存しない（メモリ上で処理→API送信→破棄） | ローカルPCに保存 |
| **文字起こし** | 全文をデータベースに保存 | ローカルPCにテキストファイル保存 |
| **要約結果** | データベースに保存 | ローカルPCにテキストファイル保存 |
| **紹介状・サマリー** | データベースに保存 | ローカルPCにテキストファイル保存 |
| **オフライン動作** | 不可 | 可能（録音のみ） |
| **フットスイッチ** | Chromeフォーカス中のみ | グローバルホットキー対応 |

**Webアプリを選択する理由:**
- ブラウザがあればどのPCからでもアクセス可能
- インストール不要、アップデートが容易
- マルチテナント（複数クリニック）対応が容易
- データの一元管理によるバックアップ・分析が容易

**Webアプリの制約:**
- ローカルファイルシステムへのアクセスはブラウザのセキュリティ制限で困難
- フットスイッチはChromeがフォーカス中のみ動作（バックグラウンド不可）
- インターネット接続必須

### 1.3 VoiceRecorderからの技術流用

| 機能 | VoiceRecorder実装 | TranslatorApp流用方針 |
|------|------------------|---------------------|
| 音声認識（STT） | Azure Fast Transcription API | ✅ 完全流用（言語識別機能を追加） |
| 要約生成（LLM） | Azure OpenAI GPT-4o | ✅ 完全流用（プロンプト調整のみ） |
| チャンク処理 | 無音検知+時間ベース分割 | ✅ ロジック流用（フロント側実装） |
| リトライ機構 | 指数バックオフ（最大3回） | ✅ 完全流用 |
| 接続プール | SocketsHttpHandler設定 | ✅ バックエンド側で流用 |
| ウォームアップ | DNS/TLS事前接続 | ✅ 流用 |
| ファイル保存 | ローカルPC保存 | ❌ **使用しない**（DB保存に変更） |
| グローバルホットキー | RegisterHotKey | ❌ **使用しない**（KeyboardEventに変更） |

### 2.5 言語判定とロール振り分けロジック

```python
# language_role_mapper.py

class LanguageRoleMapper:
    """
    音声から検出された言語をロール（医師/患者）にマッピングする
    """
    
    def __init__(self, doctor_language: str, patient_language: str):
        """
        Args:
            doctor_language: 医師の言語コード (例: "ja-JP")
            patient_language: 患者の言語コード (例: "en-US")
        """
        self.doctor_language = doctor_language
        self.patient_language = patient_language
        
        # 言語→ロールのマッピング
        self.language_to_role = {
            doctor_language: "doctor",
            patient_language: "patient"
        }
        
        # ロール→翻訳先言語のマッピング
        self.role_to_target_language = {
            "doctor": patient_language,   # 医師の発言 → 患者の言語に翻訳
            "patient": doctor_language     # 患者の発言 → 医師の言語に翻訳
        }
    
    def get_role(self, detected_language: str) -> str:
        """
        検出された言語からロールを判定
        
        Args:
            detected_language: Azure STTが検出した言語コード
        
        Returns:
            "doctor" or "patient" or "unknown"
        """
        # 完全一致を試行
        if detected_language in self.language_to_role:
            return self.language_to_role[detected_language]
        
        # 言語ファミリーで判定（en-US と en-GB は同じ英語として扱う）
        detected_base = detected_language.split('-')[0]
        for lang, role in self.language_to_role.items():
            if lang.split('-')[0] == detected_base:
                return role
        
        return "unknown"
    
    def get_translation_target(self, role: str) -> str:
        """
        ロールから翻訳先言語を取得
        
        Args:
            role: "doctor" or "patient"
        
        Returns:
            翻訳先の言語コード
        """
        return self.role_to_target_language.get(role, self.doctor_language)
```

```python
mapper = LanguageRoleMapper(
    doctor_language=request.doctor_language,
    patient_language=request.patient_language

definition = {
    "locales": [request.doctor_language, request.patient_language],

    "translated_text": translated,
    "detected_language": detected_lang,
```

#### 🔴 POC-1: Chrome上でのフットスイッチ対応（最重要）

**検証目的:**
VoiceRecorderではグローバルホットキー（`RegisterHotKey`）を使用していたが、Webアプリでは使用不可。
Chrome上でフットスイッチからの入力を受け取れるかを検証する。

**検証項目:**

| 項目 | 検証内容 | 合格基準 |
|------|---------|---------|
| KeyboardEvent | フットスイッチが標準キーボードイベントとして認識されるか | 任意のキーコードで`keydown`イベントが発火 |
| バックグラウンド動作 | Chromeがフォーカスを失った状態でも動作するか | ×（ブラウザの制限で不可能と予想） |
| フォーカス中動作 | Chromeがフォーカス中にフットスイッチで録音開始/停止できるか | ボタンクリックと同等の操作が可能 |
| WebHID API | HIDデバイスとして直接アクセス可能か（代替案） | デバイス一覧にフットスイッチが表示 |

#### 🟠 POC-2: Web Audio APIでの録音

**検証目的:**
ブラウザでマイク録音ができるか、VoiceRecorderと同等の音質（16kHz, 16bit, モノラル）が得られるか

**検証項目:**

| 項目 | 検証内容 | 合格基準 |
|------|---------|---------|
| マイク許可 | ユーザー許可でマイクアクセス可能か | `getUserMedia`成功 |
| サンプルレート | 16kHzでの録音が可能か | 16000Hz設定が反映 |
| WAV変換 | WebM/OpusをWAVに変換できるか | Azure STTが受理可能 |

**検証コード:**

#### 🟡 POC-3: Azure Fast Transcription API（多言語対応）

**検証目的:**
VoiceRecorderで使用しているFast Transcription APIが複数言語（日本語+英語など）を同時認識できるか

**検証項目:**

| 項目 | 検証内容 | 合格基準 |
|------|---------|---------|
| 多言語locale指定 | `locales: ["ja-JP", "en-US"]`が受理されるか | HTTP 200 |
| 言語識別 | 日本語音声→`ja-JP`、英語音声→`en-US`と識別されるか | 正しい言語コードが返却 |
| 中国語対応 | `zh-CN`を追加しても動作するか | HTTP 200 |

**検証コード（Python）:**
```python
# poc_azure_stt.py

import httpx
import asyncio
import os

AZURE_SPEECH_KEY = os.environ["AZURE_SPEECH_KEY"]
REGION = "japaneast"
ENDPOINT = f"https://{REGION}.api.cognitive.microsoft.com/speechtotext/transcriptions:transcribe?api-version=2024-11-15"

async def test_multilingual_transcription(audio_path: str, locales: list[str]):
    """
    多言語対応のFast Transcription APIをテスト
    """
    print(f"\n=== テスト: locales={locales} ===")
    
    async with httpx.AsyncClient(timeout=120.0) as client:
        with open(audio_path, "rb") as f:
            audio_bytes = f.read()
        
        # multipart/form-data
        files = {
            "audio": (os.path.basename(audio_path), audio_bytes, "audio/wav"),
        }
        data = {
            "definition": f'{{"locales":{locales},"format":"Display"}}'
        }
        headers = {
            "Ocp-Apim-Subscription-Key": AZURE_SPEECH_KEY,
            "Ocp-Apim-Subscription-Region": REGION,
        }
        
        response = await client.post(ENDPOINT, files=files, data=data, headers=headers)
        
        print(f"Status: {response.status_code}")
        
        if response.status_code == 200:
            result = response.json()
            print(f"✅ 成功")
            if "combinedPhrases" in result and result["combinedPhrases"]:
                text = result["combinedPhrases"][0].get("text", "")
                print(f"Text: {text[:100]}...")
            if "phrases" in result and result["phrases"]:
                detected_locale = result["phrases"][0].get("locale", "unknown")
                print(f"Detected locale: {detected_locale}")
        else:
            print(f"❌ 失敗: {response.text}")
        
        return response.status_code == 200

async def main():
    # テスト用音声ファイル（日本語と英語をそれぞれ用意）
    test_cases = [
        # (音声ファイル, テストするlocale設定)
        ("test_japanese.wav", ["ja-JP"]),
        ("test_english.wav", ["en-US"]),
        ("test_japanese.wav", ["ja-JP", "en-US"]),  # 多言語
        ("test_english.wav", ["ja-JP", "en-US"]),   # 多言語
        ("test_japanese.wav", ["ja-JP", "en-US", "zh-CN"]),  # 3言語
    ]
    
    results = []
    for audio_path, locales in test_cases:
        if os.path.exists(audio_path):
            success = await test_multilingual_transcription(audio_path, locales)
            results.append((audio_path, locales, success))
        else:
            print(f"⚠️ ファイル未存在: {audio_path}")
    
    print("\n=== 検証結果サマリー ===")
    for audio, locales, success in results:
        status = "✅ Go" if success else "🔴 No-Go"
        print(f"{status}: {audio} + {locales}")

if __name__ == "__main__":
    asyncio.run(main())
```

### 7.2 データストレージ戦略

| データ種別 | 保存先 | 保存タイミング | 備考 |
|-----------|--------|---------------|------|
| **音声データ** | なし（メモリのみ） | - | プライバシー保護、ストレージ節約 |
| **文字起こし（チャンク単位）** | ConversationLogs | チャンク処理完了時 | 原文＋翻訳をセットで保存 |
| **診察セッション情報** | Encounters | 診察開始時 | 言語設定、患者名など |
| **SOAP形式カルテ** | Encounters.SOAP_Note | 要約完了時 | 全文字起こしから生成 |
| **紹介状・サマリー** | Encounters.Referral_Letter | 生成完了時 | SOAP_Noteから派生生成 |
| **API使用量** | BillingLogs | API呼び出し時 | STT/翻訳/GPT使用量を記録 |

### 7.3 VoiceRecorderとの比較

| 項目 | VoiceRecorder（ネイティブ） | TranslatorApp（Web） |
|------|---------------------------|---------------------|
| 音声ファイル | `C:\temp\recording_*.wav` | **保存しない** |
| 文字起こし | `C:\temp\recording_*.txt` | `ConversationLogs`テーブル |
| 要約結果 | `C:\temp\recording_*.summary.txt` | `Encounters.SOAP_Note` |

### 7.4 セッション途中でブラウザを閉じた場合の対応

Webアプリでは、ブラウザを閉じるとメモリ上の音声データは失われます。
ただし、**チャンク単位でDBに保存**しているため、途中までの文字起こしは保持されます。


### 7.5 beforeunload警告の実装

ユーザーが誤ってブラウザを閉じないよう、録音中は警告を表示します。

```typescript
// useBeforeUnload.ts
import { useEffect } from 'react';

export function useBeforeUnload(isRecording: boolean) {
  useEffect(() => {
    const handleBeforeUnload = (e: BeforeUnloadEvent) => {
      if (isRecording) {
        e.preventDefault();
        e.returnValue = '録音中です。ページを離れると現在のチャンクの音声データが失われます。';
        return e.returnValue;
      }
    };

    window.addEventListener('beforeunload', handleBeforeUnload);
    return () => window.removeEventListener('beforeunload', handleBeforeUnload);
  }, [isRecording]);
}
```

## 8. 今回のまとめ

### 計画書v2.1での主な内容

1. **Webアプリ専用の計画書として明確化**
   - 音声ファイルは保存しない（メモリ上で処理→API送信→破棄）
   - 文字起こし・要約結果はすべてデータベースに保存
   - ローカルファイルシステムには依存しない設計

2. **言語・ロール設定機能**
   - 医師の言語と患者の言語を個別に設定可能

   - テナント（クリニック）ごとにデフォルト言語を設定可能
   - `LanguageRoleMapper`クラスで言語とロールのマッピングを管理

3. **技術検証優先型の開発フェーズ**
   - **Phase 0（POC）を最初に配置** → 全技術の実現可能性を個別検証
   - 特に**POC-1: フットスイッチ対応**を最重要項目として設定
   - Go/No-Go判定表で技術的リスクを明確化
   - **POC-1がNo-Goの場合 → ネイティブアプリ計画書に移行**

4. **セッション中断時のデータ復旧設計**
   - チャンク単位でDB保存するため、途中までの文字起こしは保持される
   - beforeunload警告でユーザーに注意喚起
